# Golf Charity App

A world-class web application built with **Next.js 15**, **Supabase**, and **Mantine UI**. The platform allows golfers to submit their Stableford scores, support their favorite charities, and participate in a monthly prize draw based on their "Rolling 5" verified scores.

## 🚀 Features

### User Experience
- **Authentication**: Secure login and signup via Supabase Auth with email verification.
- **Rolling 5 Scores**: Users track their 5 most recent golf scores. New entries automatically replace the oldest ones to maintain a rolling window.
- **Scorecard Verification**: Ability to upload images/screenshots of scorecards for admin verification.
- **Charity Integration**: Users can select which charity their contributions support via a dynamic charity picker.
- **Winnings Overview**: A dedicated dashboard section to view total prize money won and payment status (Pending/Completed).
- **Subscription Management**: Integration with **Razorpay** for monthly subscriptions. Includes a **Dummy Payment** mode for development and testing.

### Admin Dashboard
- **User Management**: Overview of player profiles and their subscription status.
- **Verification Queue**: A centralized system for admins to review scorecard evidence and approve/reject submitted scores.
- **Monthly Draw Engine**:
    - **Logic Configuration**: Random selection algorithm for winning numbers (1-45).
    - **Simulations**: Run "dry-run" draws to see potential winner counts and prize distributions across tiers.
    - **Publishing**: Formalize results, calculate prize splits, and automatically record winners.
- **Reports & Analytics**:
    - Total registered users.
    - Total cumulative prize pool.
    - Charity contribution totals (50/50 revenue split model).
- **Winner Management**: Full list of draw winners with the ability to mark payouts as completed.

## 🛠 Tech Stack

- **Framework**: Next.js 15 (App Router)
- **Styling**: Mantine UI (v7), Tailwind CSS, Lucide Icons
- **Backend/Database**: Supabase (PostgreSQL, Row Level Security, RPC functions, Storage)
- **Payments**: Razorpay SDK & Webhooks
- **Language**: TypeScript

## ⚙️ Setup Instructions

### 1. Environment Variables
Create a `.env.local` file in the root directory:

```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_role_key

RAZORPAY_KEY_ID=your_razorpay_key
RAZORPAY_KEY_SECRET=your_razorpay_secret
RAZORPAY_WEBHOOK_SECRET=your_webhook_secret
RAZORPAY_PLAN_ID=your_subscription_plan_id
```

### 2. Database Schema
The application requires the following tables in Supabase:
- `profiles`: User data, subscription status, and admin flags.
- `golf_scores`: Stableford entries with verification flags and image URLs.
- `charities`: Directory of available charities.
- `draw_results`: Historical record of winning numbers and pools.
- `winners`: Links users to draws with prize amounts and payout status.
- `dummy_payments`: Logs for test transactions.

### 3. Installation
```bash
npm install
npm run dev
```

## 🏆 Prize Draw Logic

The draw compares a player's **5 verified scores** against **5 winning numbers** (1-45) generated by the system:

| Match Count | Prize Tier | Pool Distribution |
| :--- | :--- | :--- |
| 5 Matches | Jackpot | 40% of Pool |
| 4 Matches | Tier 2 | 35% of Pool |
| 3 Matches | Tier 3 | 25% of Pool |

## 🔒 Security

- **Row Level Security (RLS)**: Users can only view their own scores and winnings.
- **Admin Protection**: Server-side checks on all `/admin` routes to ensure the `is_admin` flag is set on the user's profile.
- **Signature Verification**: Razorpay webhooks are validated using HMAC-SHA256 to prevent spoofing.

## 📝 Notes
- **Daily Limit**: Only one score entry is permitted per date.
- **Editing**: Existing entries for a specific date must be edited or deleted before a new one can be added for that same day.
- **Eligibility**: Users must have an `active` subscription and 5 `verified` scores to be included in the draw calculation.
